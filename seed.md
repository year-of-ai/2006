# Repository DNA

## 1. Concept Definition

> Machine-readable. Every generic skill, prompt, and agent reads this block to know what to grow.
> Change the `subject` (and re-derive `taxonomy`) to retarget the entire framework at a new concept.

```yaml
concept:
  subject: "the year 2006"
  scope: "Events, people, works, and developments that occurred in or are primarily associated with the calendar year 2006."
  taxonomy:
    - name: "History & Politics"
      slug: "history-politics"
    - name: "Science & Technology"
      slug: "science-technology"
    - name: "Arts & Culture"
      slug: "arts-culture"
    - name: "Society & Economics"
      slug: "society-economics"
    - name: "People"
      slug: "people"
  source_strategy: "Verify every fact against at least two authoritative sources: one encyclopedic (Wikipedia, Britannica) and one specialist (period newspaper archives, scholarly articles, official records) where possible."
  conventions:
    knowledge_table: "## Notable Events of 2006"
    file_path: "<category-slug>/<topic-slug>.md"
    frontmatter:
      - title
      - date
      - category
    tone: "Factual, neutral, encyclopedic. Third person. No editorial commentary."
```

## 2. Identity & Mission

This repository is a **self-growing knowledge base** about its `concept.subject` (currently **the
year 2006**). It started from a single prompt and expands autonomously: each growth tick researches
new on-topic content, builds structure (indices, timeline, cross-references), verifies facts, and
publishes ‚Äî all driven by the Concept Definition above. The framework is **concept-agnostic**:
point it at a different subject and it grows a different knowledge base. 1987 is the reference
instance; 2006 is the current instance, spawned from the year 2005.

## 6. Growth Loop

`/grow` runs one tick via the **architect** agent:
1. **Orient** ‚Äî read this seed, `ROADMAP.md`, `README.md`, repo tree.
2. **Plan** ‚Äî `plan-roadmap` skill picks the next 1‚Äì3 items (content / structure / meta).
3. **Execute** ‚Äî content ‚Üí `curator`; structure ‚Üí `build-structure`; self-improve (periodic) ‚Üí `evolve`.
4. **Verify** ‚Äî concept scope + `source_strategy`; no duplicate rows; valid links; required frontmatter.
5. **Record** ‚Äî update `ROADMAP.md`; `sync-seed` (regenerate sections 1‚Äì7); `encode-seed` (append to log).
6. **Publish** ‚Äî `publish-session` commits and pushes to `main`.

Forward plan / backlog lives in [ROADMAP.md](ROADMAP.md). Run unattended via a Claude scheduled
routine (`/schedule` ‚Üí `/grow`).

## 7. Rebuild Procedure

To reconstruct this repository from scratch using only this seed:
1. Read the **Concept Definition** (section 1).
2. Run `/genesis "<concept.subject>"` (or recreate the files in section 3 ‚Äî they are concept-agnostic
   and identical across instances).
3. `genesis` writes this Concept Definition into a fresh `seed.md`, ensures the `.github/` layer
   exists, creates `README.md` framed for the subject with a starter knowledge table, and seeds
   `ROADMAP.md` from the taxonomy.
4. Run `/grow` repeatedly (or via schedule) to regrow content and structure.
Because `sync-seed` keeps sections 1‚Äì7 current, this procedure rebuilds whatever state the repo had
reached, not just the original.
